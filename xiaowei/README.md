##1. task3[踩过的坑]
###1)浮动，
    位置：中间放最后，左右栏的位置随意
        浮动会脱离当前所在的文档流，根据前面元素的位置决定浮动的位置。
        它的浮动的上面位置是和它前一个原来在标准流中的元素底部对应，而不是包裹它的外面的元素。
###2）定位，
    位置：没有要求
        子没办法撑起父级，高度塌陷。min-height
        [一般我们不使用position来对页面内部的元素进行具体的布局，这样容易导致问题且扩展性低]
###3)margin
    位置:中间在前面即可
    原理:所有元素都浮动,改变浮动位置
    坑:中间的min-width和左中右合起来的min-width
